# 01 — Excel Pivot Automation Engine

> Turn a raw multi-channel campaign CSV into a formatted, multi-sheet Excel report in seconds. **Python + pandas + openpyxl.**

**Self-directed portfolio project.** The demo data is synthetic and labeled as such.

## Problem

Analysts and marketers are routinely handed raw campaign exports — one row per channel × segment × day — and asked for the same summaries every time: spend and ROAS by channel, conversion rate by segment, the monthly trend, the top and bottom performers. Building those by hand in Excel is repetitive, slow, and easy to get subtly wrong — especially when different people define "ROAS" or "conversion rate" differently.

## Data

A **synthetic**, seeded dataset of ~1,600 rows: 6 months (Jan–Jun 2025), 4 channels (Email, Paid Search, Social, Display), 4 customer segments (New, Returning, VIP, Dormant).

| column | meaning |
|--------|---------|
| `date` | campaign day |
| `channel` | marketing channel |
| `segment` | customer segment |
| `spend` | ad spend ($) |
| `impressions` | times an ad was shown |
| `clicks` | clicks received |
| `conversions` | completed conversions |
| `revenue` | revenue generated ($) |

Generated by [`generate_demo_data.py`](generate_demo_data.py) and documented in [`data/README.md`](data/README.md). It is **not real** — it's shaped to be realistic so there's something genuine to analyze.

## Method

[`pivot_engine.py`](pivot_engine.py) reads the CSV, computes the summaries with pandas, and writes a formatted workbook with openpyxl. Five sheets:

1. **KPI Summary** — one-page headline numbers + a "how each metric is defined" block.
2. **Revenue & ROAS by Channel**
3. **Conversion Rate by Segment**
4. **Month-over-Month Trend** (incl. month-over-month revenue change)
5. **Top-Bottom 5 Performers** — channel × segment combinations ranked by ROAS.

ROAS cells are shaded **green** (≥ 1.0, profitable) or **red** (< 1.0, underwater). The metric definitions are baked into the workbook so a reviewer always sees how each number was computed:

- **ROAS** = revenue / spend
- **CTR** = clicks / impressions
- **Conversion rate** = conversions / clicks
- **CPA** = spend / conversions
- **Revenue per conversion** = revenue / conversions

The calculation functions are pure and unit-tested (`tests/`); the Excel formatting is separate and smoke-tested end-to-end.

## How to run

```bash
# from this folder (01-excel-pivot-engine/)
pip install -r requirements.txt
python generate_demo_data.py             # writes data/demo.csv (already committed)
python pivot_engine.py data/demo.csv     # writes output/campaign_summary.xlsx
```

Runs in well under 30 seconds on the demo data. Point it at any CSV with the same columns:

```bash
python pivot_engine.py your_file.csv -o your_report.xlsx
```

### Running the tests (optional)

```bash
pip install -r requirements-dev.txt
pytest tests -v
```
