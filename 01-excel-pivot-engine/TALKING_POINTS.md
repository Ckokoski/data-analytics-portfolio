# Talking Points — Excel Pivot Automation Engine

A 2-minute interview walkthrough (STAR), plus three likely follow-ups. Tool facts are filled in; anything marked `>>> CHRISTOPHER` is yours to complete **in your own words** — your findings and decisions, not the tool's.

## 2-minute walkthrough (STAR)

**Situation.** Marketing and analytics teams constantly get handed raw campaign exports — one row per channel, per segment, per day — and they need the same summaries every time: spend and ROAS by channel, conversion rate by segment, the month-over-month trend, the best and worst performers. Doing that by hand in Excel is repetitive and error-prone.

**Task.** I wanted a tool that turns any campaign CSV into a clean, formatted, multi-sheet Excel report in seconds — and I built it as a self-directed project to sharpen my Python while solving a problem straight out of my marketing-technology background.

**Action.** I wrote a Python CLI: pandas for the math, openpyxl for the formatting. It produces a one-page KPI summary plus four breakdown sheets, with conditional formatting that flags any ROAS under 1.0 in red. I deliberately separated the pure calculation functions from the Excel-writing code so the math is unit-tested independently, and I baked every metric definition into the workbook so nobody has to guess how a number was computed. I also wrote a seeded generator that produces realistic but clearly-labeled synthetic data, so the whole thing runs out of the box.

**Result.** `>>> CHRISTOPHER:` This is the part interviewers care about most — what the data actually told you. Run the tool, then say, in your own words: which channel was underwater, which segment was the standout, and the one or two decisions you'd push a marketing team to make. Keep it to 2–3 sentences.

## Likely follow-up questions

**1. "Is this real data?"**
Honest answer: the dataset is **synthetic** — generated by a seeded script, clearly labeled, with no real company's numbers in it. I shaped it to behave realistically so there's something genuine to analyze.
`>>> CHRISTOPHER:` optionally add one line about your *real* marketing-data experience (GA4, Looker Studio, SFMC reporting) so they know you've done this for real, too.

**2. "Why Python instead of just doing pivot tables in Excel?"**
Repeatability. A pivot table is manual every time and easy to get subtly wrong; this runs the same way on any file with one command, with the metric definitions baked in so two people can't compute ROAS three different ways.
`>>> CHRISTOPHER:` add roughly how often you'd have used something like this in your prior role.

**3. "Walk me through how it actually works."**
CSV in → pandas groups and aggregates → five summary tables → openpyxl writes and formats the workbook (number formats, frozen headers, ROAS color rules). Pure functions do the math and are unit-tested; the CLI just wires it together.
`>>> CHRISTOPHER:` pick one function in `pivot_engine.py` now that you can comfortably explain, so it isn't cold if they ask you to go deeper.
